
# AU Healthcare Centre Management System

AU Healthcare Centre Management System is a web-based application designed to digitalize and automate healthcare center operations.  
It streamlines patient registration, doctor consultation, pharmacy, lab, and ambulance workflows using a role-based system.  
The project is built with a modern frontend and a backend API to support secure and efficient healthcare management.

---

## 📂 Project Structure

```

AU-Healthcare-Centre/
│── Frontend/   # React (Vite) frontend
│── Backend/    # Node.js + Express backend
│── README.md

````

---

## 🚀 Getting Started

### 1️⃣ Clone the Repository
```bash
git clone https://github.com/UdhyaKumarKMIT/AU-Healthcare-Centre.git
````

```bash
cd AU-Healthcare-Centre
```

---

## 🖥️ Frontend Setup (React + Vite)

### Navigate to Frontend

```bash
cd Frontend
```

### Install Dependencies

```bash
npm install
```

### Start Development Server

```bash
npm run dev
```

📍 Frontend will run at:

```
http://localhost:5173
```

---

## ⚙️ Backend Setup (Node.js + Express)

### Navigate to Backend

```bash
cd Backend
```

### Install Dependencies

```bash
npm install
```

### Start Backend Server

```bash
npm run dev
```

*or*

```bash
node server.js
```

📍 Backend will run at:

```
http://localhost:5000
```

## 👥 User Roles Supported

* Patient (Student / Staff / Staff Relation)
* Receptionist
* Doctor
* Nurse
* Lab Technician
* Pharmacist
* Specialist Doctor
* Ambulance Driver

---

## 🔐 Key Features

* Role-based login and dashboards
* Patient registration and medical history
* Doctor consultation and prescription management
* Pharmacy medicine dispensing workflow
* Lab test assignment and report handling
* Specialist appointment booking
* Ambulance trip logging
* Secure API-based communication

---

## 🛠️ Technologies Used

* Frontend: React (Vite), HTML, CSS, JavaScript
* Backend: Node.js, Express.js
* Database: MySQL (planned)
* Version Control: Git & GitHub

---

## 📌 Notes

* `node_modules` are ignored using `.gitignore`
* Ensure Node.js **20.19+ or 22+** is installed
* Backend and Frontend must be run in separate terminals
